# 엑셀 견적서 가져오기 기능 — 설계

> 작성일: 2026-06-12 · 상태: 승인됨 (구현 전)

## 목적

솜씨인테리어가 쓰는 엑셀 견적서(.xls)를 웹사이트에 업로드하면 웹 견적서(Estimate)로
자동 변환한다. **잘못된 금액이 손님에게 나가면 안 되므로 신뢰성이 최우선** — 검증에
실패하면 저장하지 않고 명확한 한국어 오류를 보여준다.

## 입력 양식 분석 (실측: `/Volumes/Somssi/견적/2026`, 41개 파일)

모든 파일이 동일한 표 구조를 사용한다 (헤더 장식만 2가지 변형):

- 상단 메타: 일자(엑셀 날짜 수치), 현장명, 평형, 고객명
- 표 헤더 행: `번호 | 공 종 | 적용부위, 제품명, 규격 | 단위 | 수량 | 단 가 | 금 액 | 비 고`
- 본문: 공종 칸(1열)이 채워진 행에서 새 공종 블록 시작. 블록 끝에 `소계:` 행
- 하단: `소계[페이지합계]`, `공과잡비`(%), `VAT`(%), `총공사금액`, `특이사항`(여러 행)
- 특수 케이스: 1열에 공종이 아닌 메모가 적힌 행 존재(예: "줄눈은 케라폭시인가요?") —
  공종 판별은 휴리스틱이 아닌 "직전 소계 행 이후 첫 항목 행" 규칙으로 처리

## 변환 규칙 (사용자 결정 사항)

| 엑셀 | 웹 (EstimateItem / EstimateData) |
|------|------|
| 공종 | `category` |
| 적용부위·제품명 | `name` |
| 단위/수량/단가 | `unit` / `quantity` / `unitPrice` (빈 값은 0) |
| 비고 | `note` |
| 인건비 행 | **일반 항목 그대로** (labor 구조로 변환하지 않음) |
| 금액 0원 행 | **전부 가져옴** (옵션 항목 보존) |
| 공과잡비 | 금액이 있으면 "공과잡비" 일반 항목으로 추가 (miscRate는 DB 미저장이므로) |
| VAT 행 | `includeVat` / `vatRate` |
| 특이사항 | `notes` |
| 소계/페이지합계/총공사금액 행 | 항목으로 넣지 않고 **검증에만 사용** |

## 아키텍처

- **파서**: `src/lib/excel-estimate.ts` — SheetJS(`xlsx`)로 .xls 파싱 → `ParsedEstimate`
  (메타 + items + 검증결과). 순수 함수로 작성해 단독 테스트 가능.
- **API**: `POST /api/estimates/import` — multipart 업로드, 파싱+검증 후 **저장하지 않고**
  미리보기 JSON 반환. `POST /api/estimates/import/commit` — 미리보기 확인 후 실제 저장
  (`mode: newProject | overwrite`, overwrite 시 projectId 필수). 기존 세션 인증
  (`getSessionUser`) 그대로 사용.
- **UI**:
  - 대시보드 "엑셀 견적서 가져오기" 버튼 → 새 프로젝트 생성(현장명 자동) + 견적서 등록
  - 견적서 페이지 "엑셀에서 가져오기" 버튼 → 현재 견적서 덮어쓰기 (덮어쓰기 경고 표시)
  - 공통 미리보기 화면: 공종별 항목·금액·총액 표시 → 사용자가 "등록" 눌러야 저장

## 신뢰성 — 3중 검증

1. **공종별 소계 대조**: 파싱한 항목 합계 = 엑셀 `소계:` 셀 값. 불일치 시 해당 공종명과
   양쪽 금액을 표시하고 업로드 거부.
2. **총액 대조**: 항목 전체 합 = 엑셀 `소계[페이지합계]`/`총공사금액` 셀. 불일치 시 거부.
3. **미리보기 필수**: 자동 저장 금지. 검증을 통과해도 사용자가 미리보기를 확인하고
   등록 버튼을 눌러야 DB에 기록.

검증 불가능한 상황(헤더 행을 못 찾음, 시트 없음, 지원 안 되는 파일)은 행 번호와 이유를
포함한 한국어 메시지로 중단한다. 부분 저장은 없다 — 전부 성공 또는 전부 거부.

## 테스트

- **골든 테스트**: 실제 41개 파일 전부에 대해 파서 실행 → 공종별 소계·총액 검증이
  모두 통과해야 함. 항목 수·총액을 스냅샷으로 고정.
- 단위 테스트: 빈 행, 0원 행, 메모가 1열에 있는 행, 공과잡비 유/무, VAT 유/무,
  헤더 변형 2종.
- API 테스트: 인증 없음 거부, 잘못된 파일 거부, overwrite 시 기존 데이터 교체 확인.

## 비범위 (YAGNI)

- .xlsx 신양식, 다중 시트, 엑셀로 내보내기(역방향), 이미지/서식 보존
- 기공/조공 labor 구조 자동 인식 (명시적으로 제외 결정됨)
